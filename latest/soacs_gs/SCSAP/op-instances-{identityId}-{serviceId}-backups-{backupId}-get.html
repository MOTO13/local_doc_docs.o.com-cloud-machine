Get a Service Backup

get

/api/v1.1/instances/{identityId}/{serviceId}/backups/{backupId}

Retrieves an existing backup of a service instance.
  • Request
  • Response
  • Examples

Request

Path Parameters
backupId
Type: string
Required: true
ID of the backup to be retrieved.
Query Parameters
includeLinkedDetails
Type: boolean
Includes linked details if set to true.
Default Value 
false

Properties

Title: DbTag schema.
This attribute is included only if the database was included in the backup.
ID for the Oracle Cloud Database Service database files. You can use this ID if you need to restore the database. For more information about restoring the Oracle Cloud Database Service, see Restoring from a Specific Backup in Using Oracle Database Cloud - Database as a Service.
expirationDate
Type: null
Title: ExpirationDate schema.
Date and time that the backup will be deleted automatically from storage. If set to null, the backup is retained indefinitely or until you manually delete it.
forceScaleInRequiredForRestore
Type: string
Title: forceScaleInRequiredForRestore
Message that indicates the current Oracle SOA Cloud Service instance has Managed Servers configured that are not included in this backup. This value is set to the following error message that provides more information:
JAAS-BR-105: Unable to restore configuration data for managed servers that are not part of the backup. The affected managedsServers are: [managedserver_x, managedserver_y]. Scale in the instance to remove these managed servers' nodes and then try to restore the instance again.
Before you restore the Oracle SOA Cloud Server instance using this backup, perform one of the following steps:
Scale in the service instance to remove the Managed Servers noted in the error message above.
To automatically scale in the Oracle SOA Cloud Service instance, set the 'forceScaleIn' attribute to true when restoring using this backup, as described in Restore a Service Instance
full
Type: boolean
Title: Full schema.
Boolean flag that specifies whether this is a full (true) or incremental (false) backup.
Full backups contain all of the service instance files, including runtime artifacts and files that change infrequently or do not change.
Incremental backups contain only the runtime artifacts of each managed VM in the service instance.

Examples

The following example shows how to view detailed information for a backup for an Oracle SOA Cloud Service instance by submitting a GET request on the REST resource using cURL. For more information, see Use cURL.

See also Initiate an On-Demand Backup of an Oracle SOA Cloud Service Instance.

cURL Command

curl -i -X GET -u joe@example.com:Welcome1! -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://api-host/paas/service/soa/api/v1.1/instances/ExampleIdentityDomain/ExampleInstance/backups/1390442367140
Note:: This example uses the URL prefix for the United States ( soacs). For information about the URL prefixes for other regions of the world, see Send Requests.

Example of Response Header

The following shows an example of the response header.

HTTP/1.1 200 OK
Date: Thu, 04 Feb 2015 16:23:56 GMT
Transfer-Encoding: chunked
Content-Type: application/json
