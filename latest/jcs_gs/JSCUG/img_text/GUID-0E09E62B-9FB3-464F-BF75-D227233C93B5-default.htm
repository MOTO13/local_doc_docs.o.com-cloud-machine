The Oracle Java Cloud Service (JCS) VM topology diagram shows a single service instance JCS1, which is comprised of 4 VMs. The WLS Admin Server and first Managed Server are located in VM1. The second Managed Server is located in VM2. These two Managed Servers are part of a cluster. The Oracle Traffic Director (OTD) Admin Server and the first OTD instance are located in VM3. Another OTD instance is located in VM4. VM1 and VM2 are connected to two Oracle Database Cloud Service deployments: DBCS1 and DBCS2. DBCS1 contains the Oracle Schema and DBCS2 contains the Application Database. The diagram shows 3 different roles accessing the components of this service instance: Administrators, Application Clients and Service Console/REST/CLI. Application Clients connect to the OTD VMs using port 443. Administrators connect to all VMs using port 22. They also connect to the OTD Admin Server using port 8989 and the WLS Admin Server using port 7002. The Service Console/REST/CLI users connect to the JCS and DBCS using port 443.
