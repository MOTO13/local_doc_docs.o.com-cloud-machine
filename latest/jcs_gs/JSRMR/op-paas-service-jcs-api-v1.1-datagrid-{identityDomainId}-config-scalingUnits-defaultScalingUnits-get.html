View Default Capacity Units

get

/paas/service/jcs/api/v1.1/datagrid/{identityDomainId}/config/scalingUnits/defaultScalingUnits

Returns a set of capacity units that can be used to provision Oracle Java Cloud Service-Coherence instances. The capacity units provide different default combinations of cache capacity and VM processing power configurations that fit most application and workflow requirements.

On Oracle Public Cloud, see About Default Capacity Units for Oracle Java Cloud Service-Coherence Instances in Using Oracle Java Cloud Service for information about the capacity unit configurations you can use. On Oracle Cloud Machine, see About Default Capacity Units for Oracle Java Cloud Service-Coherence Instances in Using Oracle Java Cloud Service on OPCM

Note: The term scaling unit used in the REST API means the same as capacity unit.

Nested Schema : scalingUnitDefs
Properties
scalingUnit
Type: object scalingUnit
Groups the definition details for a default capacity unit.
scalingUnitName
Type: string
Valid names include:
  • BASIC
  • SMALL
  • MEDIUM
  • LARGE

Each default capacity unit comprises a group of VMs with a predefined compute shape (processing power and RAM), running a number of JVMs or Managed Coherence Servers to provide a preconfigured primary cache size. The amount of primary cache storage to allocate for Coherence is based on the general rule of splitting the JVM heap size into thirds using: 1/3 for primary cache storage, 1/3 for backup storage, and 1/3 for scratch space.

On Oracle Public Cloud, see About Default Capacity Units for Oracle Java Cloud Service-Coherence Instances in Using Oracle Java Cloud Service to help you decide which default configuration to use. On Oracle Cloud Machine, see About Default Capacity Units for Oracle Java Cloud Service-Coherence Instances in Using Oracle Java Cloud Service on OPCM.

Nested Schema : scalingUnit
Type: object
Groups the definition details for a default capacity unit.
Properties
heapSize
Type: string
Heap size to configure with each JVM.
jvmCount
Type: integer
Number of JVMs to start on each VM for the given shape.
maxHeap
Type: string
Maximum heap that can be supported for the given capacity unit and data center site.
maxPrimary
Type: string
Maximum primary cache storage that the service instance can be scaled out to for the given capacity unit and data center site.
shape
Type: string
Compute shape for all VMs to be provisioned. For example, oc3. A shape defines the number of Oracle Compute Units (OCPUs) and the amount of RAM (system memory) available for a VM instance.

On Oracle Public Cloud, see About Machine Images and Shapes in Using Oracle Compute Cloud Service (for PaaS) for more information about shapes.

totalHeap
Type: string
Total amount of heap provided by a capacity unit.
totalPrimary
Type: string
Total amount of primary cache provided by a capacity unit. This amount is 1/3 of total heap.
vmCount
Type: integer
Number of VMs to configure for a capacity unit.

Examples

The following example shows how to view information about all default capacity units for an Oracle Java Cloud Service account by submitting a GET request on the REST resource using cURL.

Note: The command in this example uses the URL structure https://rest_server_url/resource-path, where rest_server_url is the Oracle Public Cloud or Oracle Cloud Machine REST server to contact for your identity domain or tenant name. See Send Requests for the appropriate URL structure to use on Oracle Public Cloud or Oracle Cloud Machine.

cURL Command

curl -i -X GET -u username:userPassword1! -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/service/jcs/api/v1.1/datagrid/ExampleIdentityDomain/config/scalingUnits/defaultScalingUnits
