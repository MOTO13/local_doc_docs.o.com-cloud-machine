Scale In a Cluster

delete

/paas/service/jcs/api/v1.1/instances/{identityDomainId}/{serviceId}/servers/{name}

Removes a Managed Server, essentially scaling in the Oracle Java Cloud Service instance by one node.

This section is applicable to scaling in the application tier only, if you are scaling a service instance that has Oracle Coherence enabled. To scale in the Coherence data tier, see Scale a Coherence Data Tier.

Scaling a cluster is not supported by Oracle Java Cloud Service - Virtual Image instances.

Scaling a cluster is not supported by Oracle Java Cloud Service instances based on WebLogic Server Standard Edition.

On Oracle Public Cloud: If you are scaling in a service instance that uses an Oracle Exadata database, when the Managed Server node is removed, the IP reservation associated with the node is released but not deleted.

Request

Path Parameters
identityDomainId
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

name
Type: string
Required: true
Name of the Managed Server to remove.
serviceId
Type: string
Required: true
Name of the Oracle Java Cloud Service instance.
Query Parameters
force
Type: boolean
Required: true
Flag that specifies whether you want to force the removal of the Managed Server, regardless of whether there are processes running. This value defaults to false.
Header Parameters
X-ID-TENANT-NAME
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

Examples

The following example shows how to scale in by one node an Oracle Java Cloud Service instance by submitting a DELETE request on the REST resource using cURL.

Note: The command in this example uses the URL structure https://rest_server_url/resource-path, where rest_server_url is the Oracle Public Cloud or Oracle Cloud Machine REST server to contact for your identity domain or tenant name. See Send Requests for the appropriate URL structure to use on Oracle Public Cloud or Oracle Cloud Machine.

cURL Command

curl -i -X DELETE -u username:userPassword1! -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/service/jcs/api/v1.1/instances/ExampleIdentityDomain/ExampleInstance/servers/ExampleI_server_4

Example of Response Header

The following shows an example of the response header:

HTTP/1.1 202 Accepted
Connection: close
Date: Thu, 29 Jan 2015 21:11:01 GMT
Transfer-Encoding: chunked
Location: https://rest_server_url/paas/service/jcs/api/v1.1/instances/ExampleIdentityDomain/servers/history/job/20
Content-Type: application/json
