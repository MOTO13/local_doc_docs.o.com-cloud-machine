Rollback a Patch

put

/paas/api/v1.1/instancemgmt/{identityDomainId}/services/jaas/instances/{serviceId}/patches/{rollbackId}/rollback

Rolls back a patch for an Oracle Java Cloud Service instance.
  • Request
  • Response
  • Examples

Request

Supported Media Types
  • application/json
Path Parameters
identityDomainId
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

rollbackId
Type: string
Required: true
ID of the rollback that is associated to the completed patching operation. To retrieve the rollback ID, get the list of applied patches.
serviceId
Type: string
Required: true
Name of the Oracle Java Cloud Service instance.
Header Parameters
X-ID-TENANT-NAME
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

Body Parameter
The request can contain a note about the rollback operation.
Root Schema : rollback-request
Properties
additionalNote
Type: string
Free-form text to provide additional information about the rollback request.

Response

Supported Media Types
  • application/json
202 Response
Accepted. See Status Codes for information about other possible HTTP status codes.
Body
The response body includes information about the rollback operation.
Root Schema : rollback-response
The response body includes information about the rollback operation.
Properties
details
Type: object details
Groups the details of the rollback operation.
status
Type: string
Status of the operation. When first initiated, the status is set to New. The status may transition to one of the following values: Completed and Cannot Complete.
Nested Schema : details
Type: object
Groups the details of the rollback operation.
Properties
jobId
Type: string
Job ID for the rollback operation. You can use the ID to review the status of the rollback operation.
message
Type: string
System message that describes the operation.

Examples

The following example shows how to roll back a patch operation for an Oracle Java Cloud Service instance by submitting a PUT request on the REST resource using cURL.

Note: The command in this example uses the URL structure https://rest_server_url/resource-path, where rest_server_url is the Oracle Public Cloud or Oracle Cloud Machine REST server to contact for your identity domain or tenant name. See Send Requests for the appropriate URL structure to use on Oracle Public Cloud or Oracle Cloud Machine.

cURL Command

curl -i -X PUT -u username:userPassword1! -d @rollback.json -H "Content-Type:application/json" -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/api/v1.1/instancemgmt/ExampleIdentityDomain/services/jaas/instances/ExampleInstance/patches/88992/rollback

Example of Request Body

The following shows an example of the request body.

{
   "additionalNote": "Rolling back the JDK patch."
}

Example of Response Body

The following shows an example of the response body in JSON format. 

{
   "details":
   {
      "jobId":"6553026",
      "message":"JCS-PATCHING-038: Rollback of service from patch [otd_jdk1.7.0_95] is submitted as an asynchronous job."
   },
   "status":"Completed"
}
