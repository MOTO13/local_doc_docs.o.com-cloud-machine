Send Requests

Review the following information before sending requests using the Oracle Java Cloud Service REST API.

URL Structure (Oracle Public Cloud)

Access the Oracle Java Cloud Service REST resources using the URL structure:

https://region-prefix.oraclecloud.com/resource-path
Where:
  • region-prefix is the URL prefix based on region. Valid values include:

    • United States: jaas

      For example: https://jaas.oraclecloud.com/resource-path

    • EMEA (Europe, the Middle East, and Africa): jcs.emea

      For example: https://jcs.emea.oraclecloud.com/resource-path

    To find the region value of your URL prefix:

    1. Sign in to the My Services dashboard for your identity domain.

    2. From the Customize Dashboard dropdown menu, choose Java.

    3. Locate the REST Endpoint field, which shows the URL to the REST server starting with your region prefix.

  • resource-path : Relative path that defines the REST endpoint. For a complete list of REST endpoints, see All REST Endpoints.

URL Structure (Oracle Cloud Machine)

Get the complete URL structure for the Oracle Java Cloud Service REST API from your Oracle Cloud Administrator or Tenant Administrator.

Access the Oracle Java Cloud Service REST resources using the URL structure:

https://my_api_url/resource-path
Where:

  • my_api_url: Host name. Get the host name (which may or may not include a port number) from your Oracle Cloud Administrator or Tenant Administrator.

  • resource-path : Relative path that defines the REST endpoint. For a complete list of REST endpoints, see All REST Endpoints.

Supported Methods

Perform basic CRUD operations (create, read, update, and delete) on your Oracle Java Cloud Service instances using standard HTTP method requests.

This table lists the methods for managing Oracle Java Cloud Service instances using the REST API.
HTTP Method Description
GET Retrieve information about an Oracle Java Cloud Service instance or a service instance component, for example, a Managed Server.
POST Create, stop, start, scale out, backup, or restore an Oracle Java Cloud Service instance.
PUT Delete an Oracle Java Cloud Service instance or update a service instance, for example, scale up a node or apply a patch.
DELETE Delete an Oracle Java Cloud Service instance component, for example, delete a backup or Managed Server node.

Supported Headers

The Oracle Java Cloud Service REST API supports headers that may be passed in the header section of an HTTP request or response.

This table lists the standard and custom headers that can be passed when using the REST API to manage Oracle Java Cloud Service instances.
Header Description and Example

Accept-Language

Set of natural languages that are preferred as a response to a request.

Examples:

Accept-Language:ja

Accept-Language:da, en-gb;q=0.8, en;q=0.7

Content-Type

Media type of the request body. Required for POST and PUT requests.

Examples:

Content-Type:application/json

Content-Type:application/vnd.com.oracle.oracloud.provisioning.Service+json

Content-Type:application/vnd.com.oracle.oracloud.datagrid.Service+json

Content-type:multipart/form-data

X-ID-TENANT-NAME

Identity domain ID or tenant name of the Oracle Java Cloud Service instance, must be used for authentication.

Examples:

X-ID-TENANT-NAME:ExampleIdentityDomain

X-ID-TENANT-NAME:myTenantName
