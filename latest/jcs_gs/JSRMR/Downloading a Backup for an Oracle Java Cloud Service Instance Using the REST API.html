Downloading or Archiving a Backup for a Service Instance

When you download a backup for an Oracle Java Cloud Service instance, you move the backup from the cloud storage container (on Oracle Public Cloud) or the NFS remote storage disk (on Oracle Cloud Machine) to block storage that is on the Administration Server. When you archive a backup for a service instance, you move the backup from block storage on the Administration Server VM to the cloud storage container (on Oracle Public Cloud) or the NFS remote storage disk (on Oracle Cloud Machine).

Note: The cURL command examples use the URL structure https://rest_server_url/resource-path, where rest_server_url is the Oracle Public Cloud or Oracle Cloud Machine REST server to contact for your identity domain or tenant name. See Send Requests for the appropriate URL structure to use on Oracle Public Cloud or Oracle Cloud Machine.

To download a backup for an Oracle Java Cloud Service instance:

  1. Review all the backups that are currently defined to retrieve the ID of the backup you want to download. 
    curl -i -X GET -u username:userPassword1! -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/api/v1.1/instancemgmt/ExampleIdentityDomain/services/jaas/instances/ExampleInstance/backups

    Specify the following options on the cURL command line:

    • -i option to include the HTTP header in the output. This option is optional.

    • -X option to indicate the type of request (GET).

    • -u option to specify the user name and password for the Oracle Java Cloud Service account for authentication.

    • -H to define a custom header, X-ID-TENANT-NAME, to identify the identity domain ID or tenant name.

    For more information, see View All Backups.

  2. Create a JSON document, download.json, that defines the details of the download operation.

    The following shows an example of the request document in JSON format. In this example, the "local" attribute is set to true.

    For more information about the request attribute, see Move a Backup (Download or Archive).

    {
    "local": true
}
  3. Download the backup, passing the JSON document created in step 2. 
    curl -i -X POST -u username:userPassword1! -d @download.json -H "Content-Type:application/json" -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/api/v1.1/instancemgmt/ExampleIdentityDomain/services/jaas/instances/ExampleInstance/backups/1449345407188

    Specify the following options on the cURL command line:

    • -i option to include the HTTP header in the output. This option is optional.

    • -X option to indicate the type of request (POST).

    • -u option to specify the user name and password for the Oracle Java Cloud Service account for authentication.

    • -d option to identify the request document, in JSON format, on the local machine (download.json).

    • -H to identify the content type as application/json and define a custom header, X-ID-TENANT-NAME, to identify the identity domain ID or tenant name.

    For more information about the REST resource, see Move a Backup (Download or Archive).

    The following shows an example of the response body in JSON format, including the backup URI that you can use to view the backup.

    {
    "operationName":"move-backup",
    "target_uri":"https:\/\/rest_server_url\/paas\/service\/jcs\/api\/v1.1\/instances\/ExampleIdentityDomain\/ExampleInstance\/backups\/1449345407188",
    "job_id":"8326"
}
  4. View detailed information about the backup using the backup URI (target_uri) returned in the response body. 
    curl -i -X GET -u username:userPassword1! -H "-ID-TENANT-NAME:ExampleIdentityDomain"https://rest_server_url/paas/api/v1.1/instancemgmt/ExampleIdentityDomain/services/jaas/instances/ExampleInstance/backups/1449345407188

    Optionally, you can include information about related backups by specifying the ?includeLinkedDetails query parameter.

  5. To archive the backup, create a JSON document, archive.json, that defines the details of the archive operation.

    The following shows an example of the request document in JSON format. In this example, the "local" attribute is set to false.

    {
    "local": false
}
  6. Archive the backup, passing the JSON document created in step 5. 
    curl -i -X POST -u username:userPassword1! -d @archive.json -H "Content-Type:application/json" -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/api/v1.1/instancemgmt/ExampleIdentityDomain/services/jaas/instances/ExampleInstance/backups/1443035291288

    Specify the following options on the cURL command line:

    • -i option to include the HTTP header in the output. This option is optional.

    • -X option to indicate the type of request (POST).

    • -u option to specify the user name and password for the Oracle Java Cloud Service account for authentication.

    • -d option to identify the request document, in JSON format, on the local machine (archive.json).

    • -H to identify the content type as application/json and define a custom header, X-ID-TENANT-NAME, to identify the identity domain ID or tenant name.

    "job_id":"8327"
}
