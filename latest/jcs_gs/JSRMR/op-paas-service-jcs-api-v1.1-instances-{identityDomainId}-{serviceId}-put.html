Delete a Service Instance

put

/paas/service/jcs/api/v1.1/instances/{identityDomainId}/{serviceId}

Deletes an Oracle Java Cloud Service instance.

On Oracle Public Cloud, if the service instance uses an Oracle Exadata database, the IP reservations associated with the Managed Server nodes are released but not deleted.

  • Response

Request

Supported Media Types
  • application/vnd.com.oracle.oracloud.provisioning.Service+json
Path Parameters
identityDomainId
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

serviceId
Type: string
Required: true
Name of the Oracle Java Cloud Service instance.
Header Parameters
X-ID-TENANT-NAME
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

Body Parameter
The request body defines the details of the delete service instance request.
Root Schema : delete-putrequest
Properties
dbaName
Type: string
Required: true
User name for the database administrator.
dbaPassword
Type: string
Required: true
On Oracle Public Cloud, this is the database administrator password that was specified when the Database Cloud Service database deployment was created.

On Oracle Cloud Machine, this is the password for the database administrator.

forceDelete
Type: boolean
Flag that specifies whether you want to force the removal of the service instance even if the database instance cannot be reached to delete the database schemas. If set to true, you may need to delete the associated database schemas manually on the database instance if they are not deleted as part of the service instance delete operation. The default value is false.
skipBackupOnTerminate
Type: boolean
Flag that specifies whether you want to back up the service instance or skip backing up the instance before deleting it. The default value is true (that is, skip backing up).

Response

Supported Media Types
  • application/json
202 Response
Accepted. See Status Codes for information about other possible HTTP status codes.
Body
The response body contains information about the delete operation.
Root Schema : delete-response
The response body contains information about the delete operation.
Properties
auto_update
Type: boolean
Flag that specifies whether updates to the Oracle Cloud Tools are automatically applied to the Oracle Java Cloud Service instance during the maintenance window. The Oracle Cloud Tools are used to manage the lifecycle of your service instance.
compliance_status
Type: string
Status indicating whether the version of Oracle Cloud Tools is out of compliance. Oracle uses Oracle Cloud Tools to manage the lifecycle of your service instance. If the Oracle Cloud Tools are out of compliance, this attribute is set to one of the following status values:
  • NEW_VERSION: Indicates a new version of Oracle Cloud Tools is available. Oracle strongly recommends that you apply this update as soon as possible.
  • DEPRECATED: Indicates that the current version of Oracle Cloud Tools is deprecated. Apply the latest Oracle Cloud Tools update to avoid any disruption of service in the future.
  • UNSUPPORTED: Indicates that the current version of Oracle Cloud Tools is not supported. Apply the latest patch to resume normal operations.

If the Oracle Cloud Tools are up-to-date, this attribute is blank.

compliance_status_desc
Type: string
Description that provides more details about the compliance status of the Oracle Cloud Tools, used to manage the lifecycle of your Oracle Java Cloud Service instance. If the service instance is out of compliance, this attribute is set to one of the following descriptions, based on the status value:If the Oracle Cloud Tools are out of compliance, this attribute is set to one of the following status values:
  • NEW_VERSION: A newer version of Oracle tools latestVersion is available. This update includes critical fixes to Oracle Cloud Tools. Oracle uses cloud tools to manage lifecycle of your service. Oracle strongly recommends that customers apply this update as soon as possible.
  • DEPRECATED: This service is currently in a deprecated state because the Oracle tools version deprecatedVersion is deprecated. Apply the latest Oracle tools update as this version may not be supported in the future.
  • UNSUPPORTED: This service is currently in an unsupported state because the Oracle tools version unsupportedVersion is obsolete. Apply the latest patch to resume normal operations.

If the Oracle Cloud Tools are up-to-date, this attribute is blank.

created_by
Type: string
Name of the account used to create the Oracle Java Cloud Service instance.
creation_time
Type: string
Date and time the Oracle Java Cloud Service instance was created.
description
Type: string
Free-form text that provides additional information about the service instance.
error_status_desc
Type: string
Error status that describes the reason that the Oracle Java Cloud Service instance is in an erroneous state. The following provides an example of the description:

This service is currently in an erroneous state as the tools are in an inconsistent state. Reason - error details. Apply the latest tools patch to resume normal operations on this service.

identity_domain
Type: string
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

last_modified_time
Type: string
Date and time the Oracle Java Cloud Service instance was last modified.
service_name
Type: string
Name of Oracle Java Cloud Service instance.
service_uri
Type: string
Resource URL of the Oracle Java Cloud Service instance.
status
Type: string
Flag that specifies the status of the Oracle Java Cloud Service instance at the time the delete operation was requested. Valid values include: Running, In Progress, Terminating, and Failed.
version
Type: string
Oracle Fusion Middleware software version. Valid values include: 12cRelease212, 12cRelease2, 12cR3 and 11g.
wlsVersion
Type: string
Oracle WebLogic Server software version. For example: 12.2.1.2.0, 12.2.1.0.x, 12.1.3.0.x and 10.3.6.0.x.

Examples

The following example shows how to delete an Oracle Java Cloud Service instance by submitting a PUT request on the REST resource using cURL.

Note: The command in this example uses the URL structure https://rest_server_url/resource-path, where rest_server_url is the Oracle Public Cloud or Oracle Cloud Machine REST server to contact for your identity domain or tenant name. See Send Requests for the appropriate URL structure to use on Oracle Public Cloud or Oracle Cloud Machine.

cURL Command

curl -i -X PUT -u username:userPassword1! -d @delete.json -H "Content-Type:application/vnd.com.oracle.oracloud.provisioning.Service+json" -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/service/jcs/api/v1.1/instances/ExampleIdentityDomain/ExampleInstance

Example of Request Body

The following shows an example of the request body in JSON format.

{
    "dbaName": "mydbausername",
    "dbaPassword": "Mypassword2!",
    "forceDelete": true
}

The following shows an example of the response header.

The Location header returns the REST resource that can be used to access the status of the delete service instance request. For more information, see View the Status of a Service Instance Lifecycle Operation.
