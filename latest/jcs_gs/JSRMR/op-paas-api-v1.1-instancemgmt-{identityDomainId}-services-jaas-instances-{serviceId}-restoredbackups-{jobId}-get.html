View the Status of a Restoration Operation

get

/paas/api/v1.1/instancemgmt/{identityDomainId}/services/jaas/instances/{serviceId}/restoredbackups/{jobId}

Returns the status of the specified restoration operation of an Oracle Java Cloud Service instance.

Note: The legacy path /paas/service/jcs/api/v1.1/instances/{identityDomainId}/{serviceId}/restoredbackups is deprecated but it will continue to be supported for the near term. However, it is recommended that you migrate your applications to use the new path at your earliest convenience to minimize any disruption in the future when the deprecated URL structure is no longer supported.

Request

Path Parameters
identityDomainId
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

jobId
Type: string
Required: true
Job ID of the restoration operation. To retrieve the job ID, see View All Restoration Operations.
serviceId
Type: string
Required: true
Name of the Oracle Java Cloud Service instance.
Header Parameters
X-ID-TENANT-NAME
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

Response

Supported Media Types
  • application/json
200 Response
OK. See Status Codes for information about other possible HTTP status codes.
Body
Root Schema : restoreHistory
Properties
backupDate
Type: string
Date and time of the backup.
backupId
Type: string
Backup ID.
configDataIncluded
Type: boolean
Flag that specifies configuration data was restored (true).
databaseIncluded
Type: boolean
This attribute is only applicable to Oracle Public Cloud. Flag that specifies the database files for the associated Database Cloud Service database deployment were not included in the Oracle Java Cloud Service instance restoration operation (false). If a database backup was performed, the database files must be restored manually.
jobId
Type: string
Job ID for the restore operation.
notes
Type: string
Free-form text that provides additional information about the restoration operation.
otdIncluded
Type: boolean
Flag that specifies whether the load balancer was restored (true). If OTD is not provisioned or was not provisioned at the time of the backup, the value is false.
recoveryCompleteDate
Type: string
Date and time that the restoration operation completed. This information is not displayed for restorations that are still in progress.
recoveryStartDate
Type: string
Date and time that the restoration operation started.
restoreBinariesType
Type: string
Type of binary restoration, whether binary restoration was invoked explicitly (resetBinaries is true) or as part of a patch rollback. Valid values include: reset or rollback.

If staticDataIncluded is true, the exact static restore type is displayed.

staticDataIncluded
Type: boolean
Flag that specifies the static binary files were not included (false) or included (true) in the restoration operation.
status
Type: string
Status of the operation. Valid values include: Completed, Failed, or In Progress.
statusDetails
Type: string
History of system messages. Each message is separated by an ellipses (...).

Examples

The following example shows how to view the status of a restoration operation for an Oracle Java Cloud Service instance by submitting a GET request on the REST resource using cURL.

Note: The command in this example uses the URL structure https://rest_server_url/resource-path, where rest_server_url is the Oracle Public Cloud or Oracle Cloud Machine REST server to contact for your identity domain or tenant name. See Send Requests for the appropriate URL structure to use on Oracle Public Cloud or Oracle Cloud Machine.

cURL Command

curl -i -X GET -u username:userPassword1! -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/api/v1.1/instancemgmt/ExampleIdentityDomain/services/jaas/instances/ExampleInstance/restoredbackups/8226
