Add a Coherence Data Tier Cluster

post

/paas/service/jcs/api/v1.1/instances/{identityDomainId}/{serviceId}/clusters

Adds a storage-enabled WebLogic Server cluster for an Oracle Java Cloud Service instance that's already provisioned. A default capacity unit or custom capacity unit definition must be declared. The Coherence data tier cluster and the existing application tier cluster (which is storage-disabled by default) are associated with the Coherence cluster.

If a service instance already has a storage-enabled WebLogic Server cluster, you cannot add a second cluster of storage-enabled servers.

The initial set of Managed Coherence Servers created in the Coherence data tier cluster is based on the configuration of the capacity unit declared for the service instance, and the scalingUnitCount property as defined in the request body.

Note: The term scaling unit used in the REST API means the same as capacity unit.

Body Parameter
The request body defines the details of the add Coherence data tier request.
Root Schema : datatier-postrequest
Different attributes are required when using a default capacity unit or a custom (advanced) capacity unit.
Properties
clusterName
Type: string
Name of the storage-enabled WebLogic Server cluster to add for the service instance. The cluster name:
  • Must no exceed 50 characters.
  • Must start with a letter.
  • Must contain only alphabetical characters, underscores (_), or dashes (-).
  • Must not contain any other special characters.
  • Must be unique within the WebLogic Server administration domain.

If the cluster name is empty or null, a name is generated from the first eight characters of the Oracle Java Cloud Service instance name using the following format: first8charsOfServiceInstanceName_DGCluster. For example: ExampleI_DGCluster

clusterType
Type: string
Required: true
Must be set to DG to add a Coherence data tier cluster.
scalingUnit
Type: object scalingUnit
Required: true
Required for custom (advanced) capacity unit only.

Groups the definition details for a capacity unit.

On Oracle Public Cloud, see About Custom Capacity Units and Advanced Configuration for Oracle Java Cloud Service-Coherence Instances in Using Oracle Java Cloud Service for guidelines on defining properties for a custom capacity unit. On Oracle Cloud Machine, see About Custom Capacity Units and Advanced Configuration for Oracle Java Cloud Service-Coherence Instances in Using Oracle Java Cloud Service on OPCM.

scalingUnitCount
Type: string
Required: true
The number of capacity units to add. This value cannot be 0 (zero).

Each capacity unit provides a fixed amount of primary cache storage to allocate for Coherence, based on the capacity unit's predefined properties for VM compute shape, number of JVMs, and individual JVM heap size.

scalingUnitName
Type: string
Required: true
Required for default capacity unit only.

Valid names include:

  • BASIC
  • SMALL
  • MEDIUM
  • LARGE

Each default capacity unit comprises a group of VMs with a predefined compute shape (processing power and RAM), running a number of JVMs or Managed Coherence Servers to provide a preconfigured primary cache size. The amount of primary cache storage to allocate for Coherence is based on the general rule of splitting the JVM heap size into thirds using: 1/3 for primary cache storage, 1/3 for backup storage, and 1/3 for scratch space.

On Oracle Public Cloud, see About Default Capacity Units for Oracle Java Cloud Service-Coherence Instances in Using Oracle Java Cloud Service to help you decide which default configuration to use. On Oracle Cloud Machine, see About Default Capacity Units for Oracle Java Cloud Service-Coherence Instances in Using Oracle Java Cloud Service on OPCM.

Nested Schema : scalingUnit
Type: object
Required for custom (advanced) capacity unit only.

Groups the definition details for a capacity unit.

On Oracle Public Cloud, see About Custom Capacity Units and Advanced Configuration for Oracle Java Cloud Service-Coherence Instances in Using Oracle Java Cloud Service for guidelines on defining properties for a custom capacity unit. On Oracle Cloud Machine, see About Custom Capacity Units and Advanced Configuration for Oracle Java Cloud Service-Coherence Instances in Using Oracle Java Cloud Service on OPCM.

Properties
heapSize
Type: string
Required: true
Required for custom (advanced) capacity unit only.

Heap size to configure per JVM in a capacity unit.

Use a number from 1 GB to 16 GB. Note that the JVM heap size multiplied by the number of JVMs per VM must not exceed the available RAM for the compute shape of the VM minus 1500 MB (reserved for operating use).

The total amount of primary cache storage to allocate for Coherence is based on the general rule of splitting the JVM heap size into thirds using: 1/3 for primary cache storage, 1/3 for backup storage, and 1/3 for scratch space.

If a custom capacity unit is configured with a single VM, there might not be space to store a backup copy of the Coherence data, so the actual data available might be more than 1/3 of the JVM heap size.

jvmCount
Type: integer
Required: true
Required for custom (advanced) capacity unit only.

Number of JVMs to start on each VM.

Use a number from 1 to 8.

shape
Type: string
Required: true
Required for custom (advanced) capacity unit only.

Compute shape to use for all VMs in a custom capacity unit. For example, oc3. A shape defines the number of Oracle Compute Units (OCPUs) and amount of memory (RAM).

Valid shapes include:

  • oc3: 1 OCPU, 7.5 GB memory
  • oc4: 2 OCPUs, 15 GB memory
  • oc5: 4 OCPUs, 30 GB memory
  • oc6: 8 OCPUs, 60 GB memory
  • oc7: 16 OCPUs, 120 GB memory
  • oc1m: 1 OCPU, 15 GB memory
  • oc2m: 2 OCPUs, 30 GB memory
  • oc3m: 4 OCPUs, 60 GB memory
  • oc4m: 8 OCPUs, 120 GB memory
  • oc5m: 16 OCPUs, 240 GB memory

On Oracle Public Cloud, see About Machine Images and Shapes in Using Oracle Compute Cloud Service (for PaaS) for more information about shapes.

vmCount
Type: integer
Required: true
Required for custom (advanced) capacity unit only.

Number of VMs to configure for a custom capacity unit. Use a number from 1 to 3. Coherence high availability is achieved when you use three VMs.

Response

202 Response
Accepted. See Status Codes for information about other possible HTTP status codes.
400 Response
Bad Request. Returned if:
  • The service instance is in a state other than Running. Oracle Java Cloud Service ensures that the service instance is in a Running state before a job is created for the operation to be carried out. In this case, the following message is returned:Service is not in Running state, current state is [Current Operation State]. The service needs to be in a Running state for datagrid operations to be carried out
  • A second request is received.
  • The service level of the provisioned service instance is Oracle Java Cloud Service-Virtual Image or BASIC. The following message is returned:The service [Service Instance Name] cannot be scaled. The service level is [BASIC] while the minimum service level to support scaling is [PAAS]

Examples

The following example shows how to initiate an add data tier cluster request for an Oracle Java Cloud Service instance by submitting a POST request on the REST resource using cURL.

Note: The command in this example uses the URL structure https://rest_server_url/resource-path, where rest_server_url is the Oracle Public Cloud or Oracle Cloud Machine REST server to contact for your identity domain or tenant name. See Send Requests for the appropriate URL structure to use on Oracle Public Cloud or Oracle Cloud Machine.

cURL Command

curl -i -X POST -u username:userPassword1! -d @adddatagridcluster.json -H "Content-Type:application/vnd.com.oracle.oracloud.datagrid.Service+json" -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/service/jcs/api/v1.1/instances/ExampleIdentityDomain/ExampleInstance/clusters

Example of Request Body (Default Capacity Unit)

The following shows an example of the request body in JSON format for using a default capacity unit definition to add a Coherence data tier cluster.

{
    "clusterType"           : "DG",
    "scalingUnitCount"      : "1",
    "clusterName"           : "ExampleDGCluster",
    "scalingUnitName"       : "SMALL"
}

Example of Request Body (Custom or Advanced Capacity Unit)

The following shows an example of the request body in JSON format for using a custom capacity unit definition to add a Coherence data tier cluster.

{
    "clusterType"           : "DG",
    "scalingUnitCount"      : "1",
    "clusterName"           : "ExampleDGCluster",
    "scalingUnit"      :
    {
            "shape"        : "oc3",
            "vmCount"      : "3",
            "heapSize"     : "4G",
            "jvmCount"     : "2"
    }
}
