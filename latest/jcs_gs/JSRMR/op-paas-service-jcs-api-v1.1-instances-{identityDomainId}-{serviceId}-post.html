Stop and Start a Service Instance and Individual VMs

post

/paas/service/jcs/api/v1.1/instances/{identityDomainId}/{serviceId}

Controls the lifecycle state of a service instance and its virtual machines (VMs). Specifically, you can:
  • Stop or start an Oracle Java Cloud Service instance and all its VMs
  • Restart the Administration Server and its associated VM
  • Stop, start, or restart the load balancer and its associated VM
  • Stop, start, or restart a Managed Server and its associated VM

On Oracle Public Cloud, see About Stopping and Starting an Oracle Java Cloud Service Instance and Individual VMs in Using Oracle Java Cloud Service for complete information about what happens when an instance is stopped and started. For example, all data in the Coherence cache is lost when a service instance that has a Coherence data tier is stopped. On Oracle Cloud Machine, see About Stopping and Starting an Oracle Java Cloud Service Instance and Individual VMs in Using Oracle Java Cloud Service on OPCM.

Note: If you have an hourly billing frequency on Oracle Public Cloud, you will not be charged for the time that an Oracle Java Cloud Service instance is in stopped state.

Request

Supported Media Types
  • application/vnd.com.oracle.oracloud.provisioning.Service+json
Path Parameters
identityDomainId
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

serviceId
Type: string
Required: true
Name of the Oracle Java Cloud Service instance.
Header Parameters
X-ID-TENANT-NAME
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

Body Parameter
The request body defines the details of the service instance or VM lifecycle request.
Root Schema : stopstart-postrequest
Properties
force
Type: boolean
Flag used to force the lifecycle state request. This flag is required when you specify the name of a Managed Server and there is more than one Managed Server running on the same VM.
lbAdminHostName
Type: string
Host name of the load balancer that you want to stop, start, or restart.

Note: This value is not supported for a load balancer that is running on Oracle Fusion Middleware 11g.

lifecycleState
Type: string
Type of lifecycle state request. Valid values include:
  • stop: Stops the Oracle Java Cloud Service instance, Managed Server, or load balancer, and the associated VM.
  • start: Starts the Oracle Java Cloud Service instance, Managed Server, or load balancer, and the associated VM.
  • restart: Restarts the Administration Server, Managed Server, or load balancer, and the associated VM. For restart, you must specify serverName or lbAdminHostName to identify the Administration Server, Managed Server, or load balancer to restart.

    Only server-level restart is supported in this release. You cannot use restart to restart the service instance. To restart the service instance, issue a stop lifecycle state request followed by a start lifecycle state request.

If the service instance is stopped, you cannot stop, start, or restart the Administration Server, Managed Servers, or load balancer.

lifecycleTimeout
Type: string
Timeout (minutes) for each request. The range of valid values are 1 to 300 minutes (inclusive). The default value is 60 minutes.
serverName
Type: string
Name of the Managed Server that you want to stop, start, or restart. The lifecycle state request is performed on the VM on which the Managed Server is running. Therefore, If more than one Managed Server is running on the VM, you must set force to true in order for the lifecycle state request to be initiated. If you specify the name of the Administration Server or a Managed Server that runs on the same VM as the Administration Server, then the lifecycleState value can only be set to restart and the force flag much be set to true.

Note: This value is not supported for:

  • Managed Servers that are running on Oracle Fusion Middleware 11g.
  • Managed Servers in the Coherence data tier.

Response

Supported Media Types
  • application/json
202 Response
Accepted. See Status Codes for information about other possible HTTP status codes.

The Location header returns a REST URI that can be used to access the status of the service instance request, as described in View the Status of a Service Instance Lifecycle Operation.

Examples

The following example shows how to restart a Managed Server named managedServer1 by submitting a POST request on the REST resource using cURL.

Note: The command in this example uses the URL structure https://rest_server_url/resource-path, where rest_server_url is the Oracle Public Cloud or Oracle Cloud Machine REST server to contact for your identity domain or tenant name. See Send Requests for the appropriate URL structure to use on Oracle Public Cloud or Oracle Cloud Machine.

cURL Command

curl -i -X POST -u username:userPassword1! -d @restartServer.json -H "Content-Type:application/vnd.com.oracle.oracloud.provisioning.Service+json" -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/service/jcs/api/v1.1/instances/ExampleIdentityDomain/ExampleInstance

Example of Request Body

The following shows an example of the request body in JSON format. In this example, the VM on which the specified Managed Server is running will be restarted. Setting the force flag to true ensures that the VM will be restarted in the event there are multiple Managed Servers running on it.

{
    "lifecycleState" : "restart",
    "force" : true,
    "serverName" : "managedServer1"
}

Example of Response Header

The following shows an example of the response header.

The Location header returns the REST resource that can be used to access the status of the lifecycle state request. For more information, see View the Status of a Service Instance Lifecycle Operation.

While the lifecycle state request is in progress, you can obtain the lifecycle state job ID (138 in the example below) from the lifecycle_control_job_id attribute when viewing your service instance details, as described in View All Service Instances and View a Service Instance and then use it to view the status of the request, as described in View the Status of a Service Instance Lifecycle Operation.
