View All Service Instances

get

/paas/service/jcs/api/v1.1/instances/{identityDomainId}

Returns information about all Oracle Java Cloud Service instances. You can view the full set of details by specifying the ?outputLevel=verbose query parameter.
  • Request
  • Response
  • Examples

Request

Path Parameters
identityDomainId
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

Query Parameters
outputLevel
Type: string
Flag that when set to verbose specifies that the response should include the full set of details for all service instances.
Header Parameters
X-ID-TENANT-NAME
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

Type: string

compliance_status
Type: string
Status indicating whether the version of Oracle Cloud Tools is out of compliance. Oracle uses Oracle Cloud Tools to manage the lifecycle of your service instance. If the Oracle Cloud Tools are out of compliance, this attribute is set to one of the following status values:
  • NEW_VERSION: Indicates a new version of Oracle Cloud Tools is available. Oracle strongly recommends that you apply this update as soon as possible.
  • DEPRECATED: Indicates that the current version of Oracle Cloud Tools is deprecated. Apply the latest Oracle Cloud Tools update to avoid any disruption of service in the future.
  • UNSUPPORTED: Indicates that the current version of Oracle Cloud Tools is not supported. Apply the latest patch to resume normal operations.

If the Oracle Cloud Tools are up-to-date, this attribute is blank.

compliance_status_desc
Type: string

Examples

The following example shows how to view information about all Oracle Java Cloud Service instances by submitting a GET request on the REST resource using cURL.

Note: The command in this example uses the URL structure https://rest_server_url/resource-path, where rest_server_url is the Oracle Public Cloud or Oracle Cloud Machine REST server to contact for your identity domain or tenant name. See Send Requests for the appropriate URL structure to use on Oracle Public Cloud or Oracle Cloud Machine.

cURL Command

curl -i -X GET -u username:userPassword1! -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/service/jcs/api/v1.1/instances/ExampleIdentityDomain
