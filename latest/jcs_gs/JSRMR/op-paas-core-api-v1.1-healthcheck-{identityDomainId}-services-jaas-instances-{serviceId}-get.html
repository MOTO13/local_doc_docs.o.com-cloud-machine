Monitor a Service Instance

get

/paas/core/api/v1.1/healthcheck/{identityDomainId}/services/jaas/instances/{serviceId}

Returns health monitoring data for the specified Oracle Java Cloud Service instance.

Monitoring information is provided at the following levels:

  • Service instance
  • Component (OTD, WLS)
  • Virtual Machine (VM)
  • Server

More detailed run-time metrics are provided at the OTD and server level.

For the OTD component, the following health metrics are provided:

  • epsLast5MinAvg - Average errors per second over the last 5 minutes.
  • errorRequestCount - Total error request count since the load balancer has been up and running.
  • rtLast4MinAvg - Average response time over the last 5 minutes.
  • requestCount - Total request count since the load balancer has been up and running.
  • upsince - Time (UTC) since the load balancer has been up and running.

For each server, the following metrics are provided:

  • heapfree - Amount of heap memory in megabyes (MBs) that is free.
  • heapmax - Maximum heap size in MBs.
  • upsince - Time (UTC) since the load balancer has been up and running.
Note: Health monitoring metrics are updated on an hourly basis. As a result, the metrics might be slightly out-of-date until the next hourly refresh.

Examples

The following example shows how to monitor an Oracle Java Cloud Service instance by submitting a GET request on the REST resource using cURL.

Note: The command in this example uses the URL structure https://rest_server_url/resource-path, where rest_server_url is the Oracle Public Cloud or Oracle Cloud Machine REST server to contact for your identity domain or tenant name. See Send Requests for the appropriate URL structure to use on Oracle Public Cloud or Oracle Cloud Machine.

cURL Command

curl -i -X GET -u username:userPassword1! -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/core/api/v1.1/healthcheck/ExampleIdentityDomain/services/jaas/instances/ExampleInstance

Example of Response Header

The following shows an example of the response header.

HTTP/1.1 200 OK
Date: Thu, 17 Nov 2016 20:23:11 GMT
Transfer-Encoding: chunked
Content-Type: application/json
