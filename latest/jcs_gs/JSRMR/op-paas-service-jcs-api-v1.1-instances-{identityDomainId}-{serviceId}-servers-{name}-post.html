Scale Out a Cluster

post

/paas/service/jcs/api/v1.1/instances/{identityDomainId}/{serviceId}/servers/{name}

Adds a new Managed Server to the specified cluster, essentially scaling out the Oracle Java Cloud Service instance by one node. Use the ?createClusterIfMissing=true query parameter to create the cluster if it does not exist.

By default, the name of a Managed Server will be generated from the first eight characters of the domain name, using the following format: first8charsOfDomainName_server_n, where n starts with 1 and is incremented by 1 for each additional Managed Server to ensure each name is unique.

This section is applicable to scaling out the application tier only, if you are scaling a service instance that has Oracle Coherence enabled. To scale out the Coherence data tier, see Scale a Coherence Data Tier.

Scaling a cluster is not supported by Oracle Java Cloud Service - Virtual Image instances.

Scaling a cluster is not supported by Oracle Java Cloud Service instances based on WebLogic Server Standard Edition.

On Oracle Public Cloud: If you are scaling out a service instance that uses an Oracle Exadata database, you must use the query parameter ?ipReservation=ip_reservation_name to assign a reserved IP address to the new node. Before scaling out, you must obtain IP reservations for the Managed Servers you are going to add; you will not be able to scale out the cluster without IP reservations. See the My Oracle Support document titled How to Request Authorized IPs for Provisioning a Java Cloud Service with Database Exadata Cloud Service (MOS Note 2163568.1) to submit a request for IP reservations.

Request

Path Parameters
identityDomainId
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

name
Type: string
Required: true
Name of the cluster to which you want to add the Managed Server.
serviceId
Type: string
Required: true
Name of the Oracle Java Cloud Service instance.
Query Parameters
createClusterIfMissing
Type: boolean
Flag that indicates whether a new cluster will be created during the scale-out operation. This flag defaults to false. If not specified, the Managed Server is added to the existing cluster.
ipReservation
Type: string
Required: true
This attribute is only available on Oracle Public Cloud. It is required and valid only when scaling out a service instance that uses an Oracle Exadata database.

Name of an unused IP reservation. The pre-allocated IP address will be assigned to the new Managed Server node during the scale-out operation.

Header Parameters
X-ID-TENANT-NAME
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

Examples

The following example shows how to scale out an Oracle Java Cloud Service instance by submitting a POST request on the REST resource using cURL. Only one node is added at a time per scale out request.

Note: The command in this example uses the URL structure https://rest_server_url/resource-path, where rest_server_url is the Oracle Public Cloud or Oracle Cloud Machine REST server to contact for your identity domain or tenant name. See Send Requests for the appropriate URL structure to use on Oracle Public Cloud or Oracle Cloud Machine.

cURL Command

curl -i -X POST -u username:userPassword1! -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/service/jcs/api/v1.1/instances/ExampleIdentityDomain/ExampleInstance/servers/ExampleI_cluster

cURL Command with query parameter for IP reservation name (on Oracle Public Cloud only)

The following command with the query parameter is required when scaling out a service instance that is using an Oracle Exadata database. This is applicable to Oracle Public Cloud only.

curl -i -X POST -u username:userPassword1! -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/service/jcs/api/v1.1/instances/ExampleIdentityDomain/ExampleInstance/servers/ExampleI_cluster?ipReservation=ipres04
