View an Auto Scaling Policy

get

/paas/api/v1.1/instancemgmt/{identityDomainId}/services/JaaS/instances/{serviceId}/autoscaling/policies

This endpoint is only available on Oracle Public Cloud.

Returns the latest automatic (auto) scaling policy for an Oracle Java Cloud Service instance.

  • Request
  • Response
  • Examples

Request

Path Parameters
identityDomainId
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account.
serviceId
Type: string
Required: true
Name of the Oracle Java Cloud Service instance.
Header Parameters
X-ID-TENANT-NAME
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account.

Response

Supported Media Types
  • application/json
200 Response
OK.
Body
The response body contains the auto scaling policy details when creating, updating, or viewing the policy.
Root Schema : autoscalingpolicy-response
The response body contains the auto scaling policy details when creating, updating, or viewing the policy.
Properties
description
Type: string
Description of the auto scaling policy.
message
Type: string
For example: Rule is Active
name
Type: string
Name of the auto scaling policy. The default name is the service instance name.
policies
Type: array policies
Groups details about the auto scaling policy.
startTime
Type: string
Date and time the auto scaling policy was created or updated.
status
Type: string
State of the auto scaling policy.
version
Type: string
Version of the auto scaling policy.
Nested Schema : policies
Type: array
Groups details about the auto scaling policy.
Items
  • Type: object policy-details
Nested Schema : policy-details
Properties
alarm
Type: object alarm
Groups the conditions for a rule to trigger an alarm.
coolDownPeriod
Type: integer
The number of minutes after a scaling activity is completed before any further scaling activities can start.
maxClusterSize
Type: integer
Maximum cluster size supported by the service instance when scaling out.
minClusterSize
Type: integer
Minimum cluster size supported by the service instance when scaling in.
operation
Type: string
The scaling operation to perform when the alarm is triggered. For example: scaleOut, scaleIn
Nested Schema : alarm
Type: object
Groups the conditions for a rule to trigger an alarm.
Properties
breachedOnInstance
Type: string
Whether auto scaling is to be performed when an alarm rule is breached on all VM instances or on any of the VM instances. For example: ALL, ANY.
comparisonOperator
Type: string
Arithmetic operation to use when comparing the specified statistic and threshold values. For example: GreaterThanOrEqualToThreshold, LessThanOrEqualToThreshold
metricName
Type: string
Name of the metric set in the alarm. For example: CPUUtilization
metricUnit
Type: string
Unit of measurement for the alarm's associated metric. For example: percent
sampleCount
Type: integer
The consecutive number of times data is compared for the given metric.
sampleFrequency
Type: integer
The interval in minutes when data is compared for the given metric.
statistic
Type: string
The statistic to apply to the alarm's associated metric. For example: AVG, MIN, MAX
threshold
Type: integer
The value against which the given statistic is compared.
404 Response
Not found. This means no auto scaling policy has been defined for this service instance. It could also mean auto scaling has not been enabled in Oracle Java Cloud Service.

Examples

The following example shows how to view the auto scaling policy defined for an Oracle Java Cloud Service instance, by submitting a GET request on the REST resource using cURL.

Note:

  • The command in this example uses the URL structure https://rest_server_url/resource-path, where rest_server_url is the Oracle Public Cloud REST server to contact for your identity domain. See Send Requests for the appropriate URL structure to use for Oracle Public Cloud.

  • In the resource path, the service type ../JaaS/.. is case-sensitive.

cURL Command

curl -i -X GET -u username:userPassword1! -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/api/v1.1/instancemgmt/ExampleIdentityDomain/services/JaaS/instances/ExampleInstance/autoscaling/policies
