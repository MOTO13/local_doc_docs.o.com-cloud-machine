Import a WebLogic Server Domain Configuration

post

/paas/api/v1.1/instancemgmt/{identityDomainId}/services/jaas/instances/{serviceId}/import

This endpoint is only available on Oracle Public Cloud.

Imports an on-premises WebLogic Server domain configuration and its deployed applications into an existing Oracle Java Cloud Service instance. The service instance must have been provisioned with the appropriate AppToCloud artifacts that are uploaded to an Oracle Storage Cloud Service container.

The AppToCloud infrastructure enables you to quickly migrate existing Java applications and their supporting Oracle WebLogic Server resources to Oracle Java Cloud Service. There are certain prerequisites that must be met, and certain on-premises tasks and Oracle Cloud tasks that must be performed before you can import the domain and applications. See About AppToCloud Prerequisites and Typical Workflow for Migrating Applications to Oracle Java Cloud Service in Using Oracle Java Cloud Service.

Briefly, the mandatory tasks are:

  • Install the on-premises AppToCloud tools
  • Check the health of the on-premises domain
  • Export the on-premises domain
  • Migrate application databases to Database Cloud Service
  • Create a service instance with AppToCloud
  • Import the domain configuration and applications into the service instance
  • Recreate some domain resources, if necessary

After completing the prerequisite tasks and before you can import the domain configuration and applications, you must create a service instance with AppToCloud to associate the service instance with the exported files that were generated with the AppToCloud tools. The payload you use to create a service instance with AppToCloud is the same as that described in Create a Service Instance, with the additional mandatory customPayload parameter in the request body. You can import the domain and applications only after you have created a service instance with customPayload.

Note: Back up your service instance prior to performing an import. If the import fails, you will be able to restore the service instance to a known working state.

Examples

The following example shows how to import a WebLogic Server domain configuration from an on-premises environment into your Oracle Java Cloud Service instance that was created with AppToCloud artifacts.

Note: The command in this example uses the URL structure https://rest_server_url/resource-path, where rest_server_url is the Oracle Public Cloud REST server to contact for your identity domain. See Send Requests for the appropriate URL structure to use for Oracle Public Cloud.

cURL Command

curl -i -X POST -u username:userPassword1! -d @importcustompayload.json -H "Content-Type:application/json" -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/api/v1.1/instancemgmt/ExampleIdentityDomain/services/jaas/instances/ExampleInstance/import
