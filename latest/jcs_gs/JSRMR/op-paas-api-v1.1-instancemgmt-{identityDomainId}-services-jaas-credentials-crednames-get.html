Retrieve the SSH Key Description Using Query Parameters

get

/paas/api/v1.1/instancemgmt/{identityDomainId}/services/jaas/credentials/crednames

Returns one or more credential descriptions for the given query parameters.
  • Use the ?credname query parameter to retrieve results for a given credential name.
  • Use the ?serviceName query parameter to retrieve results for a given Oracle Java Cloud Service instance.

Note the following:

  • By default, 50 keys are returned by a given query. Use the ?limitRowCount query parameter to control the page size (number of results returned in a single request).
  • Use the ?offset query parameter to fetch additional results, if available. By default, offset is 0 (zero).

Request

Path Parameters
identityDomainId
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

Query Parameters
credname
Type: string
Unique credential name.

Note: vmspublickey is the only supported credential name. This is the default if a credential name is not supplied.

serviceName
Type: string
Name of the Oracle Java Cloud Service instance. If a service name is not supplied, the request will return all service instances for the given identity domain ID or tenant name.
Header Parameters
X-ID-TENANT-NAME
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

Response

Supported Media Types
  • application/json
200 Response
OK. See Status Codes for information about other possible HTTP status codes.
Body
The response body contains general information about one or more SSH keys (including their descriptions).
Root Schema : keydescquery-response
The response body contains general information about one or more SSH keys (including their descriptions).
Properties
canonicalUrl
Type: string
URL to query the credential key value.
credName
Type: string
Credential name. Currently, the only supported credential name is vmspublickey.
credType
Type: string
Credential type. Only supported value is SSH.
description
Type: string
Credential description of the given credential name.
hasMore
Type: boolean
Flag that specifies whether the list returned by the query is full (false) or partial (true).

When the list is partial, use the offset query parameter to fetch additional results. By default, offset is 0 (zero) and limitRowCount is 50.

identityDomain
Type: string
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

items
Type: string
Groups credential description details returned by the query.
serviceName
Type: string
Name of the Oracle Java Cloud Service instance.
serviceType
Type: string
Cloud service type. Valid value is jaas for Oracle Java Cloud Service.
totalResults
Type: integer
Number of results returned by the query.

Examples

The following example shows how to retrieve SSH key descriptions by using query parameters in a GET request on the REST resource using cURL.

Note:

  • Currently, the only supported credential name is vmspublickey.

  • The command in this example uses the URL structure https://rest_server_url/resource-path, where rest_server_url is the Oracle Public Cloud or Oracle Cloud Machine REST server to contact for your identity domain or tenant name. See Send Requests for the appropriate URL structure to use for Oracle Public Cloud or Oracle Cloud Machine.

cURL Command

curl -i -X GET -u username:userPassword1! -H "X-ID-TENANT-NAME:ExampleIdentityDomain" "https://rest_server_url/paas/api/v1.1/instancemgmt/ExampleIdentityDomain/services/jaas/credentials/crednames?credname=vmspublickey&serviceName=ExampleInstance"

Example of Response Header

The following shows an example of the response header.
