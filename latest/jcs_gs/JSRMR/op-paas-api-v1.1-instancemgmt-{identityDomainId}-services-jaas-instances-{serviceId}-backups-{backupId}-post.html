Move a Backup (Download or Archive)

post

/paas/api/v1.1/instancemgmt/{identityDomainId}/services/jaas/instances/{serviceId}/backups/{backupId}

Enables you to move a backup of an Oracle Java Cloud Service instance to:
  • Oracle Storage Cloud Service container (archive)
  • Block storage on the Administration Server (download)

On Oracle Public Cloud, see About Backup and Restoration of Oracle Java Cloud Service Instances in Using Oracle Java Cloud Service for complete information about contents of backups and where backups are stored. On Oracle Cloud Machine, see About Backup and Restoration of Oracle Java Cloud Service Instances in Using Oracle Java Cloud Service on OPCM.

Note: The legacy path /paas/service/jcs/api/v1.1/instances/{identityDomainId}/{serviceId}/backups is deprecated but it will continue to be supported for the near term. However, it is recommended that you migrate your applications to use the new path at your earliest convenience to minimize any disruption in the future when the deprecated URL structure is no longer supported.

Request

Supported Media Types
  • application/json
Path Parameters
backupId
Type: string
Required: true
Backup ID. To retrieve the backup ID, see View All Backups.
identityDomainId
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

serviceId
Type: string
Required: true
Name of the Oracle Java Cloud Service instance.
Header Parameters
X-ID-TENANT-NAME
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

Body Parameter
The request body contains details of the archive or download backup request.
Root Schema : archivedownload-request
The request body defines the details of the archive or download backup request.
Properties
local
Type: boolean
Required: true
Flag that specifies whether the backup is to be downloaded and stored locally in block storage on the Administration Server (true), or archived and stored remotely (false).

On Oracle Public Cloud, the remote storage is the specified Oracle Storage Cloud Service container.

On Oracle Cloud Machine, the remote storage is the specified NFS storage disk.

Response

Supported Media Types
  • application/json
202 Response
Accepted. See Status Codes for information about other possible HTTP status codes.
Body
The response body contains information about the archive or download backup operation.
Root Schema : archivedownload-response
The response body contains information about the archive or download backup operation.
Properties
job_id
Type: string
Job ID for the archive or download backup operation.
operationName
Type: string
Name of operation performed. Set to: move-backup.
target_uri
Type: string
URI of the archive or download backup resource.

Examples

The following example shows how to download or archive a backup for an Oracle Java Cloud Service instance by submitting a POST request on the REST resource using cURL.

Note: The command in this example uses the URL structure https://rest_server_url/resource-path, where rest_server_url is the Oracle Public Cloud or Oracle Cloud Machine REST server to contact for your identity domain or tenant name. See Send Requests for the appropriate URL structure to use for Oracle Public Cloud or Oracle Cloud Machine.

cURL Command

curl -i -X POST -u username:userPassword1! -d @movebackup.json -H "Content-Type:application/json" -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/api/v1.1/instancemgmt/ExampleIdentityDomain/services/jaas/instances/ExampleInstance/backups/1390442367140

Example of Request Body

The following example shows the contents of the request body in JSON format, for a download operation.

{
   "local":true
}

The following example shows the contents of the request body in JSON format, for an archive operation.

{
   "local":false
}
