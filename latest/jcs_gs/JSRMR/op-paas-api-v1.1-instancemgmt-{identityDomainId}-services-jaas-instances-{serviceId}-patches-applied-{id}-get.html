View a Patch Operation

get

/paas/api/v1.1/instancemgmt/{identityDomainId}/services/jaas/instances/{serviceId}/patches/applied/{id}

Returns information about the specified patching operation for an Oracle Java Cloud Service instance.
  • Request
  • Response
  • Examples

Request

Path Parameters
id
Type: string
Required: true
Unique ID of the patching operation. The ID can be retrieved by getting the list of applied patches.
identityDomainId
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

serviceId
Type: string
Required: true
Name of the Oracle Java Cloud Service instance.
Header Parameters
X-ID-TENANT-NAME
Type: string
Required: true
Identity domain ID for the Oracle Java Cloud Service account (on Oracle Public Cloud).

Tenant name for the Oracle Java Cloud Service instance (on Oracle Cloud Machine).

Examples

The following example shows how to view a patch operation for an Oracle Java Cloud Service instance by submitting a GET request on the REST resource using cURL.

Note: The command in this example uses the URL structure https://rest_server_url/resource-path, where rest_server_url is the Oracle Public Cloud or Oracle Cloud Machine REST server to contact for your identity domain or tenant name. See Send Requests for the appropriate URL structure to use on Oracle Public Cloud or Oracle Cloud Machine.

cURL Command

curl -i -X GET -u username:userPassword1! -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/api/v1.1/instancemgmt/ExampleIdentityDomain/services/jaas/instances/ExampleInstance/patches/applied/110375

Example of Response Header

The following shows an example of the response header:

HTTP/1.1 200 OK
Date: Wed, 18 May 2016 17:52:37 GMT
Transfer-Encoding: chunked
Content-Type: application/json
