Update the Backup Configuration

post

/paas/api/v1.1/instancemgmt/{identityDomainId}/services/jaas/instances/{serviceId}/backupconfig

Updates the backup configuration for an Oracle Java Cloud Service instance. Specifically, you can update the following details:
  • Schedule for full and incremental backups
  • Default retention time for incremental backups and full on-demand backups; note that full scheduled backups are retained until their last related incremental backup is no longer available, if any
  • Storage Cloud Service URI and user name for the container that is used to store backups (Oracle Public Cloud)
  • NFS URI of the remote storage that is used to store backups (Oracle Cloud Machine)

To disable a scheduled full or incremental backup, set the fullBackupSchedule or incrementalBackupSchedule value, respectively, to null. For example: {"fullBackupSchedule": null}.

To disable and renable the backup service for a service instance, use the backups parameter. When disabled, both on-demand and scheduled automated backups cannot be performed.

On Oracle Public Cloud, see About Backup and Restoration of Oracle Java Cloud Service Instances in Using Oracle Java Cloud Service for complete information about scheduled automated backups, contents of backups, and where backups are stored. On Oracle Cloud Machine, see About Backup and Restoration of Oracle Java Cloud Service Instances in Using Oracle Java Cloud Service on OPCM.

Note: The legacy path /paas/service/jcs/api/v1.1/instances/{identityDomainId}/{serviceId}/backupconfig is deprecated but it will continue to be supported for the near term. However, it is recommended that you migrate your applications to use the new path at your earliest convenience to minimize any disruption in the future when the deprecated URL structure is no longer supported.

Root Schema : backupconfig-postrequest
The request body defines the details of the backup configuration update request. There are no required attributes, but you must specify the backups parameter to disable or reenable the backup service for the specified service instance.
Properties
backups
Type: string
State of the backup service you want to set for the specified Oracle Java Cloud Service instance. Valid value: DISABLE, ENABLE, or UNCHANGED (default).

When the backup service is disabled, both on-demand and scheduled automated backups are disabled. The following operations are also not permitted on the service instance when the backup service is disabled:

  • Update the backup configuration
  • Restore from an existing backup
  • Apply a patch
  • Rollback a patch
cloudStorageContainer
Type: string
Where to store service instance backups.

On Oracle Public Cloud, this is the URI of an existing Oracle Storage Cloud Service container. Use the format storageservicename-storageidentitydomain/containername

If you change the storage container, any existing backups in the previous storage container will remain there and be available for service restoration until the retention period has elapsed.

If you try to change the storage container and the storage user credentials (cloudStorageUser and cloudStoragePwd), the storage container will not be updated to the new URI if at least one existing backup in the old container cannot be used because the old container cannot be accessed with the new credentials.

On Oracle Cloud Machine, this is the NFS URI of the remote storage. Use the format ip-address:export-path

This value cannot be changed if the NFS remote storage disk contains backup archives for the service instance.

cloudStoragePwd
Type: string
Password of the user name to access the specified container. This attribute is only available on Oracle Public Cloud.
cloudStorageUser
Type: string
Oracle Storage Cloud Service user name used to access the specified container. This attribute is only available on Oracle Public Cloud.
defaultRetention
Type: string
Number of days incremental backups and full on-demand backups are retained. This value defaults to 30. Full scheduled backups are retained until their last related incremental backup is no longer available, if any.

Any change to the default retention value will affect new backups as well as existing backups. This means if you change the value to smaller number, existing backups that meet the updated retention policy will be deleted at the next scheduled maintenance cycle.

Set the default retention value to null to retain backups indefinitely or until you manually delete them.

fullBackupSchedule
Type: array fullBackupSchedule
Schedule for the next full backup.

By default, full backups are initiated weekly starting 12 hours after an instance was created, rounded to the nearest five-minute interval.

Note: Though not recommended, you can disable full backups temporarily by setting this attribute to null.

incrementalBackupSchedule
Type: array incrementalBackupSchedule
Schedule for the next incremental backup.

By default, incremental backups are initiated every day except the day that full backups are initiated.

Note: Though not recommended, you can disable incremental backups temporarily by setting this attribute to null.

The response body includes information about the backup configuration. If specific changes in the backup configuration are requested and successfully completed, the response body shows the updated configuration. Otherwise, the response body shows the existing backup configuration for the service instance.
Root Schema : backupconfig-response
The response body includes information about the backup configuration. If specific changes in the backup configuration are requested and successfully completed, the response body shows the updated configuration. Properties
backupDestination
Type: string
Location of the backups (for PAAS service level instances only).

Note the difference between Oracle Public Cloud and Oracle Cloud Machine.

On Oracle Public Cloud, this configurable value defaults to BOTH (which means Disk and Oracle Storage Cloud Service Container).

On Oracle Cloud Machine, this non-configurable value defaults to Disk and NFS remote storage.

cloudStorageContainer
Type: string
Where service instance backups are stored.

On Oracle Public Cloud, this is the URI of the Oracle Storage Cloud Service container.

On Oracle Cloud Machine, this is the NFS URI of the remote storage.

cloudStorageUser
Type: string
User name to access the specified Oracle Storage Cloud Service container. This attribute is only available on Oracle Public Cloud.
defaultRetention
Type: string
Number of days incremental backups and full on-demand backups are retained. If set to null, backups are retained indefinitely or until you manually delete them.

Full scheduled backups are retained until their last related incremental backup is no longer available, if any.

extendedRestoreTypes
Type: string
Only value is NONE. This means that restoration of backups is supported only by using a backup ID.
fullBackupSchedule
Type: array fullBackupSchedule
Schedule for the next full backup.
incrementalBackupSchedule
Type: array incrementalBackupSchedule
Schedule for the next incremental backup.
lastBackupDate
Type: string
Date and time that the last backup completed.

Note: If there have been no backups, this attribute is omitted.

lastSuccessfulCleanupDate
Type: string
Date and time that the last successful cleanup job completed. The cleanup job is run after the scheduled backup for the day is completed. The cleanup job purges old records, deletes expired backups or removes expired local copies (that are due to be deleted or removed that day).
nextFullBackupDate
Type: string
Scheduled date and time for next full backup.

On Oracle Public Cloud, this attribute is not available when the state of the backup service is disabled for the service instance.

nextIncrementalBackupDate
Type: string
Scheduled date and time for next incremental backup.

On Oracle Public Cloud, this attribute is not available when the state of the backup service is disabled for the service instance.

percentBackupVolumeUsed
Type: string
Total percent of the backup volume that is currently in use.
scheduledBackups
Type: string
Type of scheduled backups. Only value is ALL. This means both full and incremental scheduled backups are enabled.
state
Type: string
State of the backup service for the service instance. For example, ENABLED or DISABLED.
totalBackupVolumeUsed
Type: string
Total amount (in GBs) of the backup volume that is currently in use.
totalBackupVolumeUsedInBytes
Type: string
Total amount (in Bytes) of the backup volume that is currently in use.
totalCloudStorageContainerUsed
Type: string
On Oracle Public Cloud, the total amount (in GBs) of the Storage Cloud Service container space that is currently in use.

On Oracle Cloud Machine, the total amount (in GBs) of the NFS remote storage disk that is currently in use.

totalCloudStorageContainerUsedInBytes
Type: string
On Oracle Public Cloud, the total amount (in Bytes) of the Storage Cloud Service container space that is currently in use.

On Oracle Cloud Machine, the total amount (in Bytes) of the NFS remote storage disk that is currently in use.

Examples

The following example shows how to update the backup configuration for an Oracle Java Cloud Service instance by submitting a POST request on the REST resource using cURL.

Note: The command in this example uses the URL structure https://rest_server_url/resource-path, where rest_server_url is the Oracle Public Cloud or Oracle Cloud Machine REST server to contact for your identity domain or tenant name. See Send Requests for the appropriate URL structure to use on Oracle Public Cloud or Oracle Cloud Machine.

cURL Command

curl -i -X POST -u username:userPassword1! -d @backupconfig.json -H "Content-type:application/json" -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/api/v1.1/instancemgmt/ExampleIdentityDomain/services/jaas/instances/ExampleInstance/backupconfig

Example of Request Body

The following shows an example of the request document in JSON format to change the automatic backup schedule. Include only those parameters that you want changed.

{
   "fullBackupSchedule":
   {
      "hour":"5",
      "dayOfWeek":"Sun"
   },
   "incrementalBackupSchedule":
   {
      "hour":"3"
   }
}

Note: You cannot change the backup configuration when the backup service for the specified service instance is in a DISABLED state.

Example of Request Body

The following shows an example of the request document in JSON format to disable the backup service for the specified service instance.

Note that the request payload should not include other parameters as supported in the backup configuration because you are disabling the backup service.

{
    "backups": "DISABLE"
}

The following shows an example of the request document in JSON format to enable the backup service for the specified service instance.

{
    "backups": "ENABLE"
}

Note that the request payload can include other parameters as supported in the backup configuration when you reenable the backup service.
