View All Deleted Backups

get

/paas/api/v1.1/instancemgmt/{identityDomainId}/services/jaas/instances/{serviceId}/deletedbackups

Returns information about all deleted backups for an Oracle Java Cloud Service instance.

On Oracle Public Cloud, see About Backup and Restoration of Oracle Java Cloud Service Instances in Using Oracle Java Cloud Service for complete information about contents of backups, where backups are stored, and how backups and local copies of backups are deleted automatically. On Oracle Cloud Machine, see About Backup and Restoration of Oracle Java Cloud Service Instances in Using Oracle Java Cloud Service on OPCM.

Note: The legacy path /paas/service/jcs/api/v1.1/instances/{identityDomainId}/{serviceId}/deletedbackups is deprecated but it will continue to be supported for the near term. However, it is recommended that you migrate your applications to use the new path at your earliest convenience to minimize any disruption in the future when the deprecated URL structure is no longer supported.

Properties
backupId
Type: string
Backup ID.
backupStartDate
Type: string
Date and time that the backup started.
databaseIncluded
Type: boolean
Flag that specifies whether a database backup was performed as part of the Oracle Java Cloud Service instance backup operation. This attribute is only applicable to Oracle Public Cloud. If false, a database backup was not performed. If true, a database backup was performed.

On Oracle Cloud Machine, the value is always false.

dbTag
Type: string
An RMAN tag or a timestamp. This attribute is only applicable to Oracle Public Cloud.

The attribute is included only if a database backup was performed as part of the Oracle Java Cloud Service instance backup. When the associated Database Cloud Service database deployment is hosting a single-instance database, the attribute value is the RMAN tag that identifies the specific database files backed up. When the associated Database Cloud Service database deployment is hosting an Oracle Real Application Clusters (RAC) database, the attribute value is the timestamp of the database backup.

deletedOnDate
Type: string
Date and time that the backup deletion completed.
expirationDate
Type: string
Date and time that the backup is deleted automatically from storage. If the value is null, the backup is retained indefinitely or until you manually delete it. (Value is null when keepForever was set to true for an on-demand backup.)
full
Type: boolean
Flag that specifies whether this was a full (true) or incremental (false) backup.
href
Type: string
URI of the backup resource.
initiatedBy
Type: string
User account that initiated the operation. Value is scheduled for scheduled automated operations.
jobId
Type: string
Job ID for the delete backup operation.
local
Type: boolean
Flag that specifies whether the backup is downloaded and stored locally in block storage on the Administration Server (true), or archived and stored remotely (false).

On Oracle Public Cloud, the remote storage is the specified Oracle Storage Cloud Service container.

On Oracle Cloud Machine, the remote storage is the specified NFS storage disk.

localCopy
Type: boolean
Flag that specifies whether the local copy of a backup is maintained for a backup that has been uploaded to remote storage.

On Oracle Public Cloud, the remote storage is the specified Oracle Storage Cloud Service container.

On Oracle Cloud Machine, the remote storage is the specified NFS storage disk.

notes
Type: string
Free-form text that provides additional information about the backup.
size
Type: string
Size of the backup, in MBs.
sizeInBytes
Type: integer
Size of the backup, in Bytes.
status
Type: string
Status of the backup. Valid value is Deleted.
storageContainer
Type: string
This atttribute is only available on Oracle Public Cloud.

Where service instance backups are stored.

Examples

The following example shows how to view information about all deleted backups for an Oracle Java Cloud Service instance by submitting a GET request on the REST resource using cURL.

Note: The command in this example uses the URL structure https://rest_server_url/resource-path, where rest_server_url is the Oracle Public Cloud or Oracle Cloud Machine REST server to contact for your identity domain or tenant name. See Send Requests for the appropriate URL structure to use for Oracle Public Cloud or Oracle Cloud Machine.

cURL Command

curl -i -X GET -u username:userPassword1! -H "Content-Type:application/json" -H "X-ID-TENANT-NAME:ExampleIdentityDomain" https://rest_server_url/paas/api/v1.1/instancemgmt/ExampleIdentityDomain/services/jaas/instances/ExampleInstance/deletedbackups
