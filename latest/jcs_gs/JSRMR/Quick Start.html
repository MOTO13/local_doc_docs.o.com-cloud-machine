Quick Start

Set up your environment and use the REST API to create your first Oracle Java Cloud Service instance by performing the tasks described in the following sections.

All documentation is applicable to using the REST API on Oracle Public Cloud and Oracle Cloud Machine unless otherwise indicated.

A Note About Terminology

Throughout the documentation, any reference to identity domain or {identityDomainId} in a resource path refers to the tenant name for Oracle Java Cloud Service on Oracle Cloud Machine.

If using a new Oracle Storage Cloud Service subscription, be sure to set a replication policy for your Oracle Storage Cloud Service before you create your first container. See How to Begin with Oracle Storage Cloud Service Subscriptions in Using Oracle Storage Cloud Service.

Note: If you do not create a storage container before you create your Oracle Java Cloud Service instance, you can set the boolean parameter createStorageContainerIfMissing to true in your request document and specify a container name that does not yet exist in Oracle Storage Cloud Service. This enables the provisioning system to create the container for you before it provisions the service instance.

Create database deployments in Oracle Database Cloud Service to host the:

  • Oracle Fusion Middleware component schemas required by Oracle Java Required Files (JRF)

  • (Optional) Application schemas

You can create the required schema database deployment and an application schema database deployment using Oracle Database 11g Release 2, Oracle Database 12c Release 1, or Oracle Database 12c Release 2. However, note that only Oracle Java Cloud Service instances based on WebLogic Server 12.2.1 fully support the Oracle Database 12.2 version for the database that contains the Oracle Infrastructure schemas required for Oracle Java Cloud Service.

Database Cloud Service Quick Start (tutorial)

Creating a Database Deployment in Using Oracle Database Cloud Service

Note: If using the BASIC level of Oracle Database Cloud Service with the BASIC level of Oracle Java Cloud Service, you must configure the Database Cloud Service environment before you create the database deployment. See Using a Database Cloud Service - Virtual Image Database Deployment.

Submit a request to obtain IP reservations (only if using an Oracle Exadata database)

If you plan to use an Oracle Exadata database with your Oracle Java Cloud Service instance, you must obtain IP reservations for the Managed Servers you are going to provision before creating the service instance; you will not be able to create the instance without IP reservations. See the My Oracle Support document entitled How to Request Authorized IPs for Provisioning a Java Cloud Service with Database Exadata Cloud Service (MOS Note 2163568.1) to learn how to submit a request for IP reservations.

Prerequisite More Information

Create an SSH public/private key pair

Create SSH Keys for Use with Oracle Cloud Services (tutorial)

Generating a Secure Shell (SSH) Public-Private Key Pair in Using Oracle Java Cloud Service on OPCM

Obtain connection details to a database

Contact your Oracle Cloud Administrator or Tenant Administrator for the following:

  • Connection string to a database in one of the following formats: host:port:SID or host:port/Service_Name

  • User name for the database administrator

  • Password for the database administrator

Note: Only Oracle Java Cloud Service instances based on WebLogic Server 12.2.1 fully support the Oracle Database 12.2 version for the database that contains the Oracle schemas required for Oracle Java Cloud Service.

Obtain connection details to an external backup storage disk

Contact your Oracle Cloud Administrator or Tenant Administrator for the URI to a remote networked disk share in one of the following formats:

  • host_name:absolute_path_to_remote_backup_disk

  • host_ip:absolute_path_to_remote_backup_disk

Obtain database network and public access network details

Contact your Oracle Cloud Administrator or Tenant Administrator for the following networks:

  • Public Access Network: Path to the network that will be used to access the service instance, in the format /tenant_name/public/EoIB_vNet_name.

  • Database Network: Path to the network through which the service instance will access the database that contains the Oracle Fusion Middleware component schemas. For example, /mytenant/public/IPoIB_dbaccess.
